# 動機

複数の開発プロジェクトを掛け持ちしているとき、Windows内でいろいろなツールやプロジェクトごとの情報源を参照することになるが、中には環境自体が重いこともあり、手動で切り替えていたが、これがわずらわしい。

具体例として、現在行っている開発AではWSL上でDockerを使ってアプリを起動しており、VSCodeで開発している。ドキュメントはBOXをマウントしている。開発BはローカルのSQLサーバを起動し、VSで開発している。開発CはVSCodeを使っているが、WSL上のファイルではなく、ホスト側のファイルを触っており、ターミナルを通じてサーバの中に入っての作業が多い。などなど。

# やりたいこと

プロジェクト作業をしていないときには、不要な環境が起動してないなど、各プロジェクトにおける最適環境を簡単に作り出せる状態を作りたい。

# つくるもの

##　概要
Windowsにおける作業環境を切り替えるWindowsアプリで、タスクトレイに常駐し、ワンクリックで環境を切り替えられる機能を提供。

プロファイルを切り替えると次の動作を行う
* 切り替え先の環境実行時イベントに登録されているアプリケーション、サービス、バッチを実行
* 切り替え元の環境終了時イベントに登録されているアプリケーション、サービス、バッチを実行

## 詳細

プロファイルの切り替えは直ちに起きるのではなく必ずダイアログで確認を取ってから切り替える

タスクトレイにはプロファイルのリストが表示されるがその中に、プロファイル編集ができるメニューがあり、新規追加や編集を行うダイアログが立ち上がる。

プロファイルには

* 環境実行時イベント
* 環境終了時イベント
* 環境用壁紙


などを登録できる。環境実行/終了時イベントは

* アプリケーションの実行
* コマンドの実行
* サービスの開始/終了

などをひとつのアクションとし、複数を登録できる

アクションには簡単な「実行前条件チェック」も登録でき、起動していたらなにもしない、終了していたら何もしない、という条件をコマンドベースで書くことができる。コマンドの戻り値や結果の正規表現を使って成功失敗を判定して実行の条件とする。

「実行後チェック」で別プロセスから監視し起動されたかどうかの確認して成功失敗判定を行うなども行えるとよい


アクション実行時に実行するかどうかをたずねるダイアログを出すかどうかのチェックボックスを設ける。このチェックが入っているものは必ずユーザの実行確認が入る


プロファイルを切り替えると壁紙を切り替え、いまのプロファイルがなにかがわかるようになる。
特定の画像を入れておけるが、登録されていない場合、プロファイル名を画像化したものを勝手に作成して使う


初期のバージョンとしてはクローズドで完結、API、ネットワークアクセスなどは使わない
